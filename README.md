# Reproducible Benchmarking Environment

This repository uses a `flake.nix` file to declaratively define the exact tools needed to build and benchmark this project. Anyone with the Nix package manager installed can reproduce the identical environment on any Linux machine — regardless of what is already installed on the system.

---

## Why this exists

Benchmarking results are only meaningful if the toolchain is identical across machines. Different JDK versions, different Gradle versions, or different system utilities can produce different results even on identical hardware. This flake guarantees that the Raspberry Pi and the HPC Spark cluster use bit-for-bit identical tools.

Without this, getting the environment working on a new machine typically involves:
- Manually installing the right JDK version
- Hoping the system package manager has the right version
- Debugging version mismatches when things behave differently

With this flake, the answer is always just `nix develop`.

---

## File Explanation

### `flake.nix`

```nix
{
  description = "A very basic flake";
  inputs = {
    nixpkgs.url = "github:nixos/nixpkgs?ref=nixos-unstable";
  };
  outputs = { self, nixpkgs }:
  let
    system = "aarch64-linux";
    pkgs = nixpkgs.legacyPackages.${system};
  in
  {
    devShells.${system}.default = pkgs.mkShell {
      packages = with pkgs; [
        jdk21
        gradle
        perf
        linuxPackages.cpupower
        s6-linux-utils
      ];
    };
  };
}
```

**`inputs`** — declares where packages come from. Here we use `nixpkgs`, the Nix package repository containing over 100,000 packages. The URL pins it to the `nixos-unstable` branch.

**`system = "aarch64-linux"`** — specifies the target architecture. Both the Raspberry Pi and the Spark cluster use 64-bit ARM, so this value is the same on both machines. If porting to an x86 machine, change this to `"x86_64-linux"`.

**`devShells.${system}.default`** — defines a shell environment containing exactly these tools:

| Package | Purpose |
|---|---|
| `jdk21` | Java 21 Development Kit for compiling and running Java benchmarks |
| `gradle` | Build tool for managing dependencies and running JMH benchmarks |
| `perf` | Linux performance profiling tool for low-level CPU analysis |
| `linuxPackages.cpupower` | Controls CPU frequency scaling — set to `performance` governor before benchmarking to ensure consistent clock speeds |
| `s6-linux-utils` | Process supervision and system utilities |

### `flake.lock`

Auto-generated by Nix the first time you run `nix develop`. Records the exact git commit hash of nixpkgs, ensuring that everyone gets identical package versions. **Always commit this file** — it is what makes the environment truly reproducible.

## How to Use

### Prerequisites

1. Install Nix:
   ```bash
   sh <(curl -L https://nixos.org/nix/install) --daemon
   ```

2. Enable flakes (add to `/etc/nix/nix.conf`):
   ```
   experimental-features = nix-command flakes
   ```

### Entering the environment

```bash
git clone <repo>
cd <repo>
nix develop
```

You are now in a shell with all tools available at the exact pinned versions. Your system is otherwise unaffected — nothing is installed globally.

### Running benchmarks

```bash
gradle jmh
```

### Exiting the environment

```bash
exit
```

All tools disappear from PATH. Your system is back to its normal state.

### Auto-activation (optional)

If you have `direnv` installed, the environment activates automatically when you enter the directory:

```bash
echo "use flake" > .envrc
direnv allow
```

## Porting to a New Machine

The only requirement is Nix. On the Spark cluster or any other machine:

```bash
# install nix if not present
sh <(curl -L https://nixos.org/nix/install) --daemon

# enable flakes
echo "experimental-features = nix-command flakes" | sudo tee -a /etc/nix/nix.conf

# clone and enter
git clone <repo>
cd <repo>
nix develop
```

The environment is identical to the one on the Raspberry Pi because `flake.lock` pins every package to the exact same version.

### If the target machine uses a different architecture

Change the `system` variable in `flake.nix`:

```nix
# for x86 machines
system = "x86_64-linux";

# for Apple Silicon
system = "aarch64-darwin";
```

Or support multiple architectures simultaneously:

```nix
let
  forAllSystems = nixpkgs.lib.genAttrs [ "aarch64-linux" "x86_64-linux" ];
in
{
  devShells = forAllSystems (system:
    let pkgs = nixpkgs.legacyPackages.${system};
    in {
      default = pkgs.mkShell {
        packages = with pkgs; [ jdk21 gradle perf ];
      };
    }
  );
}
```

## Before Benchmarking

Set the CPU governor to `performance` to prevent frequency scaling from affecting results:

```bash
echo performance | sudo tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor
```

Verify current frequency:

```bash
cat /sys/devices/system/cpu/cpu*/cpufreq/scaling_cur_freq
```
